# TFG_Fraud_detection_licenses_airbnb_listings
This repository contains the code and data pipeline developed for the Bachelor’s Thesis project on regulatory risk scoring in tourist rental listings in Barcelona.

The project focuses on Airbnb listings and the official tourist licence registry of Barcelona. Its objective is not to legally determine whether a listing is illegal, but to build a reproducible machine learning pipeline that estimates whether an Airbnb listing belongs to the proxy class potentially_unlicensed.

The final output is a modelling proof of concept based on data quality analysis, geospatial enrichment, model comparison, validation-based threshold selection and SHAP explainability.

## Project Overview

The project follows a Bronze, Silver and Gold data architecture:

Bronze layer: raw input datasets.
Silver layer: cleaned, standardised and anonymised datasets.
Gold layer: final model-ready dataset with selected features, proxy target and train/validation/test splits.

The modelling workflow compares several supervised models, including classical machine learning models, a TF-IDF text-based model and a simple neural network. The final selected model is a Random Forest classifier, chosen based on its balance between F1-score, PR-AUC, recall, precision and interpretability.

## Important Note About the Data

A static version of the raw Bronze datasets is included in this repository so that the results can be verified and reproduced.

Only the raw input datasets used in the Bronze layer are provided. The Silver and Gold datasets are generated by running the notebooks in order. This ensures that all intermediate datasets, modelling splits, figures and results can be reproduced from the same initial data snapshot.

The included data should be interpreted as a static academic snapshot. Listings, licences and regulations may change over time, so results should not be assumed to represent the current state of the market.

## How to Run the Project

It is recommended to run the project inside a Python virtual environment to avoid dependency conflicts with other projects.

### 1. Open the project folder

Open a terminal in the root folder of the repository:

```bash
cd Fraudulent_property_listing_detection_tool
```

### 2. Create a virtual environment

Using standard Python:

```bash
python -m venv .venv
```

Activate it on Windows:

```bash
.venv\Scripts\activate
```

Activate it on macOS/Linux:

```bash
source .venv/bin/activate
```

### 3. Install the required dependencies

Once the virtual environment is activated, install the dependencies from `requirements.txt`:

```bash
pip install -r requirements.txt
```

Alternatively, using `uv`:

```bash
uv venv
```

Activate the environment on Windows:

```bash
.venv\Scripts\activate
```

Activate the environment on macOS/Linux:

```bash
source .venv/bin/activate
```

Then install the dependencies:

```bash
uv pip install -r requirements.txt
```

### 4. Launch Jupyter

Start Jupyter from the activated environment:

```bash
jupyter notebook
```

or:

```bash
jupyter lab
```

### 5. Execute the notebooks in order

The notebooks must be executed in the following order:

1. `src/1_Data_preprocessing.ipynb`
2. `src/2_Exploratory_data_analysis.ipynb`
3. `src/3_Modelling.ipynb`
4. `src/4_Results_and_explainability.ipynb`

Each notebook depends on the outputs generated by the previous one. The Bronze raw datasets are provided as a static snapshot in `data/bronze/`, while the Silver, Gold, model and report outputs are generated by running the pipeline.
